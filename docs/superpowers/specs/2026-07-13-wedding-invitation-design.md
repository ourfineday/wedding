# 모바일 청첩장 설계 문서 (GitHub Pages)

- 작성일: 2026-07-13
- 참고: https://velog.io/@cadenzah/wedding-invitation-review , https://cadenzah.github.io/wedding/

## 1. 목표
개발자가 직접 만드는 모바일 청첩장을 GitHub Pages로 **무료 배포**하고, 카카오톡/링크로 친구들과 공유한다. 스몰웨딩 특성상 **대상에 따라 다른 정보**를 보여주는 것이 핵심이다.

## 2. 핵심 요구사항
- **두 버전 제공**
  - *공지용*(기본 링크): 예식 장소·오시는 길 **제외**
  - *초대용*(`?to=invite`): 예식 장소·오시는 길 **포함**
- 두 버전 **모두 계좌번호 없음**
- **사진 2장** (지금은 임시 placeholder, 나중에 파일만 교체)
- **D-Day 카운터**
- **반응형**(모바일 + PC 모두 편하게)
- **4가지 디자인 테마**를 실제 화면으로 미리 본 뒤 택1

## 3. 비목표 (YAGNI — 만들지 않음)
- 계좌번호, RSVP(참석 여부), 방명록, 배경음악
- 백엔드/DB (순수 정적 사이트)
- 빌드 도구(React/Vite/Node) — 불필요

## 4. 아키텍처 — 빌드 없는 정적 사이트
```
github-io/
├─ index.html          구조(뼈대) 1개
├─ styles/
│   ├─ base.css         공통 레이아웃 · 반응형
│   └─ themes.css       4가지 테마 색/폰트 (CSS 변수로 전환)
├─ scripts/
│   ├─ config.js        ★ 내용 전부 (이름·날짜·장소·사진·인사말·카카오키)
│   └─ main.js          렌더링 + D-Day + 버전분기 + 테마 + 카카오
├─ images/
│   ├─ main.jpg         임시 사진 1 (나중에 교체)
│   └─ sub.jpg          임시 사진 2 (나중에 교체)
└─ README.md            배포 · 수정 · 카카오 설정 방법
```
- 콘텐츠는 `config.js` **한 곳**에 모아 유지보수를 쉽게 한다.
- `main.js`가 `config.js`를 읽어 DOM을 렌더하고, D-Day·버전분기·테마·카카오 공유를 처리한다.

## 5. 콘텐츠 모델 (`config.js`)
- 신랑/신부 이름, (선택) 부모님 성함
- 예식 일시(ISO 문자열), 장소명·주소·좌표(위/경도)
- 인사말 텍스트
- 사진 경로 2개
- 카카오 JavaScript 키 (빈 값 허용)
- 기본 테마 이름

## 6. 두 버전 동작
- URL 파라미터 `?to=invite` → 초대용. 없으면 공지용(기본).
- 장소·오시는 길 섹션은 "비공개(private)"로 표시해두고 **초대용에서만 노출**한다.
- 카카오 공유·링크복사 버튼은 **현재 보고 있는 URL(파라미터 포함)**을 공유한다 → 초대할 친구에겐 `?to=invite` 링크, 나머지에겐 기본 링크.
- 보안 잠금이 아니라 "실수로 장소가 노출되지 않게 하는" 수준의 분리다.

## 7. 섹션 구성 (위→아래)
1. **히어로** — 사진1 + 두 사람 이름 + 예식 날짜
2. **인사말** — 초대 문구
3. **D-Day 카운터** — 예식까지 남은 날, 예식 후 자동 "D+N"
4. **일정** — 날짜·시간(달력형 표시)
5. **오시는 길** *(초대용만)* — 지도 표시 + **티맵·카카오맵·네이버지도** 3개 길찾기 버튼
6. **공유** — 카카오톡 공유 버튼 + 링크 복사 버튼

## 8. 4가지 테마 (미리보기 후 택1)
- `minimal`(미니멀·모던), `classic`(클래식·로맨틱), `natural`(내추럴·따뜻), `reference`(레퍼런스풍)
- CSS 변수로 색/폰트를 전환. 상단 **테마 선택 바** + `?theme=` 링크로 실시간 비교.
- **내용·기능은 4개 모두 동일**, 스킨(색/폰트/장식)만 다름. 택1 후 그 테마를 기본값으로 고정하고 선택 바는 숨긴다.

## 9. 지도 (오시는 길)
- **기본(키 불필요)**: 지도 이미지 1장 + 길찾기 버튼 3개.
  - 딥링크: 티맵(`tmap://`), 카카오맵(`https://map.kakao.com/link/...`), 네이버지도(`nmap://` + 웹 fallback)
- **업그레이드(선택)**: `config.js`에 카카오 JS 키가 있으면 **카카오맵 임베드(움직이는 지도)**로 자동 전환.
- 길찾기 버튼 3개는 어느 경우에도 항상 동작(개발자 가입 불필요).

## 10. 카카오톡 공유
- 카카오 JS 키가 있으면 `Kakao.Share.sendDefault`로 **카드형**(이미지 + 제목 + "청첩장 보기" 버튼) 공유.
- 키가 없으면 **링크 복사 / Web Share API**로 폴백(카톡에 링크만 붙여도 OG 미리보기는 노출).
- OG 메타태그는 **공지용 정보 기준**으로만 넣어 링크 미리보기에 장소가 노출되지 않게 한다.

## 11. 반응형
- 모바일 세로가 기준. PC에서는 중앙 카드(약 480px) + 배경 여백으로 정갈하게.

## 12. 배포 & 이후 수정
- GitHub 저장소 → **Settings ▸ Pages ▸ main 브랜치**만 켜면 배포(빌드 없음).
- 진행: 로컬에서 4테마 완성 → 사용자 택1 → GitHub 업로드 + Pages 설정 안내 → *(선택)* 카카오 키 발급/도메인 등록 안내.
- README에 "사진 바꾸는 법 / 글자 바꾸는 법 / 날짜·장소 바꾸는 법"을 명시.

## 13. 지금은 임시(placeholder)로 두는 항목
사진 2장(회색 박스), 이름·날짜·장소·인사말(예시 텍스트), 카카오 키(빈 값), 지도 좌표(임의 위치). → 전부 나중에 `config.js`·`images/`만 고치면 반영.

## 14. 이후(선택) 확장
- 카카오 JS 키 발급 → 카드형 공유 + 움직이는 지도 활성화
- 커스텀 도메인 연결(원할 경우)

## 15. 필요한 계정 요약
| 항목 | 필요 여부 |
|---|---|
| GitHub 계정 | 필수(무료) — Pages 호스팅 |
| 카카오 개발자 앱(JS 키) | 선택 — 카드형 공유 + 움직이는 지도 |
| 지도 3버튼 · 지도 이미지 | 계정 불필요 |
| 계좌/RSVP/방명록/음악 | 해당 없음 |
